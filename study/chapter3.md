[1] 브랜치란
 브랜치의 의미

‘브랜치(Branch)’는 나뭇가지(branch) 를 뜻하며,
Git에서는 하나의 프로젝트에서 여러 개의 독립된 작업 흐름을 관리하기 위한 도구이다.

즉, 브랜치를 사용하면 기존 버전(main) 은 그대로 두고
새로운 기능을 개발하거나 수정 작업을 독립적으로 진행할 수 있다.

 브랜치가 필요한 이유

여러 사람이 동시에 작업하거나, 실험적인 기능을 추가할 때
하나의 코드에서 직접 수정하면 충돌이나 손실이 발생할 수 있다.

브랜치를 사용하면 다음과 같은 장점이 있다.

기존 코드에 영향을 주지 않고 새로운 시도 가능

각 기능별로 독립적인 개발 환경 유지

완료된 후 병합(merge) 을 통해 안전하게 통합 가능

🪜 기본 브랜치(main)

Git 저장소를 만들면 자동으로 main(또는 master) 브랜치가 생성된다.

사용자가 git commit을 실행할 때마다
main 브랜치는 최신 커밋(HEAD) 의 위치를 가리킨다.

[2] 브랜치 만들기 및 이동하기
 새 브랜치 만들기
git branch          # 브랜치 목록 확인
git branch [이름]   # 새 브랜치 생성


예시:

git branch feature


➡️ feature라는 이름의 새 브랜치가 생성된다.

 현재 브랜치 확인하기

git log 실행 시 (HEAD -> 브랜치명) 으로 표시된다.

터미널 프롬프트 끝에도 (main) 등 현재 브랜치명이 나타난다.

git log --oneline   # 한 줄로 간단히 로그 확인

🔄 브랜치 전환하기

브랜치를 이동할 때는 git switch 명령을 사용한다.

git switch [이름]


예시:

git switch feature


➡️ feature 브랜치로 이동하여,
main 브랜치의 내용은 그대로 유지한 채 별도의 작업을 진행할 수 있다.

[3] 브랜치 정보 확인하기
 브랜치와 커밋 관계 확인하기
git log --branches


각 브랜치의 최신 커밋 정보를 한눈에 확인 가능.

git log --graph


브랜치 간의 관계를 그래프 형태로 시각화하여 보여준다.

분기(Branch)와 병합(Merge) 흐름을 쉽게 파악할 수 있다.

 브랜치 간 차이점 비교하기
git log [브랜치1]..[브랜치2]


➡️ 왼쪽 브랜치에는 없고, 오른쪽 브랜치에만 있는 커밋 내역을 보여준다.

예시:

git log main..feature


→ feature 브랜치에서 새로 추가된 커밋만 표시.

[4] 브랜치 병합하기
 서로 다른 파일 병합하기

1️⃣ main 브랜치로 전환

git switch main


2️⃣ 새 브랜치의 변경 내용 병합

git merge [브랜치명]


3️⃣ 결과 확인

ls -al


→ 새 브랜치에 있던 파일이 main 브랜치에도 반영되어 있음을 확인.

 서로 다른 브랜치에서 한 문서의 다른 부분을 수정한 경우

자동으로 충돌 없이 자연스럽게 병합된다.

새 브랜치의 수정 내용과 main 브랜치의 내용이 함께 반영됨.

 같은 부분을 수정한 경우 (충돌 발생)

1️⃣ main 브랜치로 전환
2️⃣ git merge [브랜치명] 실행
3️⃣ Git이 자동 병합 불가 메시지를 출력
4️⃣ 수정 파일을 열면 다음과 같은 표시가 나타남:

<<<<<<< HEAD
(main 브랜치 내용)
=======
(병합할 브랜치 내용)
>>>>>>> feature


5️⃣ 충돌 부분을 직접 수정한 뒤 다시 커밋한다.

🗑️ 병합 완료 후 브랜치 삭제
git branch -d [이름]


➡️ 병합이 완료된 브랜치를 삭제할 때 사용.
(주의: 아직 병합되지 않은 브랜치는 -D 옵션으로 강제 삭제 가능.)

 cherry-pick으로 병합하기

전체 브랜치를 합치지 않고,
특정 커밋만 선택적으로 병합할 때 사용한다.

git switch [대상브랜치]         # 병합될 브랜치로 이동
git cherry-pick [커밋 해시]     # 특정 커밋만 가져와 병합


예시:

git cherry-pick a1b2c3d


➡️ a1b2c3d 커밋만 현재 브랜치에 병합된다.

[5] 문제 풀이
번호	명령어	설명
1	git branch fixed	새 브랜치 생성
2	git switch fixed	브랜치 전환
3	git log --oneline	로그를 한 줄로 보기
4	git add .	변경된 파일 모두 스테이징
5	git log --graph	브랜치와 커밋 관계를 그래프로 보기
6	cat edit.txt	파일 내용 확인
7	git init	깃 저장소 초기화
8	git merge fixed	fixed 브랜치 병합
9	git branch -d fixed	병합 완료된 브랜치 삭제

🧾 정리 요약
항목	핵심 내용
브랜치(Branch)	프로젝트 내 독립된 작업 흐름을 관리하는 가지
기본 브랜치	main (기본 생성)
주요 명령어	branch, switch, merge, log, cherry-pick
병합 시 주의점	같은 부분을 수정하면 충돌 발생 가능
병합 후 정리	필요 없는 브랜치는 git branch -d로 삭제
